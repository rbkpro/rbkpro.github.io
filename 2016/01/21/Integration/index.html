In this post I will show you how to come up quickly and use Jersey JAX-RS with your existing application that you built using Spring stack technology:

II- Why we need that ?

Actually Jersey framework basically uses ServletContainer to intercept all the incoming requests. As we configure in our projects web.xml, that all the incoming rest request should be handled by that servlet. And to avoid the conflict with the Spring servlet you must instantiate 2 different servlet.

Another way to reach that

It worth mention that you can reach that by change Jersey servlet from :

"com.sun.jersey.spi.container.servlet.ServletContainer"

To Spring MVC servlet : 

"com.sun.jersey.spi.spring.container.servlet.SpringServlet"
